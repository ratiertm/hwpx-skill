- 넘치면: 사람이 하는 방식대로 ① 폰트 한 단계 ↓ ② 줄간격 10%씩 ↓ ③ 셀 높이 비례 ↓ 단계 적용

Step F: **컨텍스트 보존 (생략 금지)**

> 다음 세션에서 구조 재판정·재설명 없이 바로 채우기를 가능하게 하는 단계.
> Step 0-A에서 컨텍스트를 로드했든 안 했든 **항상 실행**.

**F-1. 최초 등록 (Step 0-A에서 "목록에 없음"이었을 때)**

```bash
# 1) 양식 등록 (이미 작업한 source.hwpx 사용)
pyhwpxlib template add <source.hwpx>

# 2) 메타데이터 기록 (이번 세션 판정 결과를 그대로)
pyhwpxlib template annotate <이름> \
  --description "<한 줄 설명>" \
  --page-standard [1page|free] \
  --structure-type [A|B] \
  --notes "<특이사항 — 서명 방식·보존 항목 등>"
```

**F-2. 채우기 이력 기록 (최초·재사용 공통, 매 세션)**

```bash
# 채운 데이터를 history.json에 기록
pyhwpxlib template log-fill <이름> --data '<{"key": "value", ...}>'
```

> `history.json` 최근 10건 유지. `template context` 출력 시 최상단에 표시되어
> 다음 세션에서 "이전에 홍길동으로 채웠는데 이번엔 김철수로" 같은 재사용이 가능해짐.

**F-3. 결정사항 기록 (이번 세션에서 새로 알게 된 것만)**

```bash
pyhwpxlib template annotate <이름> \
  --add-decision "<날짜 자동 삽입> <이번 세션 발견 사항>"

# 예시:
# --add-decision "서명란 셀 높이 3000 미만 시 이미지 잘림 확인"
# --add-decision "3행 colspan 병합 시 구조 B 판정 변경 불필요 재확인"
```

> 아무 새 발견이 없으면 생략 가능. 있으면 반드시 기록.

**F-4. Memory 등록 (최초 등록 시 1회만)**

```
Memory에 저장:
"hwpx 양식 [<한글 이름>] 등록 완료 (<날짜>).
다음 세션: pyhwpxlib template context <이름> 으로 즉시 복원."
```

## 설계 검토 포인트

### 1. "생략 금지" 강제가 현실적인가?

**문제**: Claude가 Step F를 빠뜨릴 수 있음.  
**대응**: On Load에서 스킬 로드 시 이미 `template list`를 실행하므로,
작업 종료 전 "Step F 실행하셨나요?" 체크를 On Load Exit 패턴으로 추가 가능.
(Stage 3에서 Claude Code hook으로 강화)

### 2. `template list`를 On Load + Step 0-A 두 번 실행하는 중복

On Load 시 결과를 캐시해서 Step 0-A에 재사용.  
실제로는 빠른 명령이라 중복 실행도 무방. 설계 단순성 우선.

### 3. Step F-2 데이터 형식 — 파일 vs 인라인 JSON

```bash
# 파일 (작업 중 임시 저장한 경우)
pyhwpxlib template log-fill <이름> --data filled_data.json

# 인라인 (바로 입력)
pyhwpxlib template log-fill <이름> --data '{"key": "value"}'
```

Stage 1 CLI 명세에서 두 방식 모두 지원하도록 설계.

### 4. 워크플로우 [1] (새 문서), [2] (편집)에는 Step F 없음

양식 채우기는 반복 사용이 전제. 새 문서·편집은 일회성이 많아 이력 축적의 ROI가 낮음.
필요하면 [1][2]에 선택적 Step F 추가 가능 (현재 설계 범위 밖).
