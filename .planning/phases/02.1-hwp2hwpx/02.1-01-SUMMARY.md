# Phase 2.1 Plan 01: Nested Table + Surrogate Pair Fix Summary

**Fixed 4 missing nested tables in HWP->HWPX by filtering record levels in _build_cell_paragraph, plus UTF-16 surrogate pair decoding for emoji/PUA characters**

## Performance

## Accomplishments
- Fixed root cause: _build_cell_paragraph iterated ALL records in paragraph group including records from nested table cells at deeper levels, which overwrote the parent paragraph's text_data/char_shape_pairs/line_segs
- ibgopongdang HWP now produces 21 tables in HWPX (was 17, 4 nested tables missing)
- Added _decode_hwp_text_chars() helper with surrogate pair combining, replacing 2 duplicated inline parsing loops
- Upgraded exception logging in _build_cell_paragraphs from debug to warning for better diagnostics
- 11 new tests covering nested table preservation, level filtering, and surrogate pair decoding

## Files Created/Modified

- `tests/test_hwp2hwpx_nested_table.py` - 11 tests: nested table e2e, unit, and surrogate pair decoding

## Decisions Made
- Root cause was record level mixing, not _collect_sub_records or _find_ctrl_headers: the for loop in _build_cell_paragraph iterated pg[1:] without filtering by level, so deeper PARA_TEXT records from nested table cells overwrote the cell paragraph's own text_data
- Fix approach: add `if rec.get('level', 0) != direct_level: continue` guard -- minimal, surgical change
- Created shared _decode_hwp_text_chars rather than adding surrogate handling to each of 3 inline loops
