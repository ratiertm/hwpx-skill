    **수정 위치**: `_build_cell_paragraph` 함수의 text_data 파싱 루프 (line 2250-2264)와 동일 패턴이 사용되는 모든 위치.

    **현재 코드 문제**:
    ```python
    while i < len(text_data) - 1:
        ch = struct.unpack_from('<H', text_data, i)[0]
        i += 2
        # ch가 0xD800-0xDBFF (high surrogate)일 때 다음 uint16을 low surrogate로 결합해야 함
        # 현재는 개별 uint16을 그대로 chr()로 변환 → U+FFFD
    ```

    **수정 방법**:
    1. `ch`가 high surrogate (0xD800 <= ch <= 0xDBFF) 범위인지 확인
    2. 맞다면 다음 2바이트를 읽어 low surrogate (0xDC00 <= low <= 0xDFFF) 확인
    3. 결합: `codepoint = 0x10000 + ((ch - 0xD800) << 10) + (low - 0xDC00)`
    4. `chr(codepoint)` 사용
    5. orphan surrogate (pair가 맞지 않는 경우) → `chr(0xFFFD)` 대체

    **주의**: 서로게이트 쌍은 4바이트(uint16 x 2)를 소비하지만 HWP의 확장 제어 문자(1-31)의 i += 14 로직에는 영향 없음 (서로게이트는 0xD800+ 범위이므로 제어 문자 범위가 아님).
