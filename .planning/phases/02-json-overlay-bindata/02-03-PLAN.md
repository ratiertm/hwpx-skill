Fix the nested table double-extraction bug in `_extract_table`:

**Current buggy code (line ~199):**
```python
for inner_tbl in tc_el.findall(f".//{_HP}tbl"):
```
This `.//{_HP}tbl` matches ALL descendant tables, so a table nested 2 levels deep gets extracted multiple times.

**Fix:** Only find direct-child tables within the cell's paragraphs, not deeper descendants. The structure is `<hp:tc> -> <hp:subList> -> <hp:p> -> <hp:run> -> <hp:tbl>`.

Replace with:
```python
# Find only immediate child tables (not deeper nested ones)
direct_tables = []
for sub_list in tc_el.findall(f"{_HP}subList"):
    for p_el in sub_list.findall(f"{_HP}p"):
        for run_el in p_el.findall(f"{_HP}run"):
            tbl = run_el.find(f"{_HP}tbl")
            if tbl is not None:
                direct_tables.append(tbl)

for inner_tbl in direct_tables:
    nested = _extract_table(
        inner_tbl, _tbl_id[0], texts, tables, images,
        _text_id, _tbl_id, _img_id,
        style_map, include_images,
    )
    nested["nested_in"] = f"tbl{tbl_idx}/r{ri}c{ci}"
    tables.append(nested)
    _tbl_id[0] += 1
```

The key insight: `_extract_table` is recursive, so each nested table will find ITS child tables when it's processed. We only need to find direct children at each level.

For tests, create inline XML with nested table structure:
```xml
<hp:tbl>
  <hp:tr><hp:tc>
    <hp:subList>
      <hp:p><hp:run><hp:tbl>  <!-- nested table -->
        <hp:tr><hp:tc>
          <hp:subList><hp:p><hp:run><hp:t>nested text</hp:t></hp:run></hp:p></hp:subList>
        </hp:tc></hp:tr>
      </hp:tbl></hp:run></hp:p>
    </hp:subList>
  </hp:tc></hp:tr>
</hp:tbl>
```

Verify that `tables` list contains exactly 2 entries (outer + 1 nested), not 3+ from double-extraction.
