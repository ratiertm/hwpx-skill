<objective>
Rewrite overlay.py extract and apply to support precise `<hp:t>` unit text replacement.

Purpose: Current `apply_overlay` uses naive `>original<` string replacement that fails when text is split across multiple `<hp:t>` elements (common in HWPX). This plan adds `original_parts` tracking in extract and regex-based multi-`<hp:t>` matching in apply, plus replaces subprocess unpack/repack with direct zipfile manipulation.

Output: Working overlay extract/apply with split-text handling, zipfile-based apply, and comprehensive tests.
</objective>
