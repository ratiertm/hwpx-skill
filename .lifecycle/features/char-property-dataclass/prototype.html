CharProperty Dataclass Expansion Prototype

Architecture: Before vs After

BEFORE (현재)
@dataclass class CharProperty: id: Optional[int] attributes: Dict[str, str] # generic child_attributes: Dict[...] # generic child_elements: Dict[...] # generic # Access pattern: height = prop.attributes.get("height") font = prop.child_attributes.get("fontRef", {}) ul_type = font.get("hangul")
→
AFTER (목표)
@dataclass class CharProperty: id: Optional[int] height: Optional[int] # typed text_color: Optional[str] # typed font_ref: Optional[CharFontRef] # typed underline: Optional[CharUnderline] bold: Optional[bool] ... # Access pattern: height = prop.height font = prop.font_ref.hangul ul_type = prop.underline.type

Data Flow

HWPX ZIP └─ header.xml └─ <hh:charPr id="0" height="1000" textColor="#000000"> ├─ <hh:fontRef hangul="0" latin="0" .../> ├─ <hh:bold/> └─ <hh:underline type="BOTTOM" shape="SOLID" color="#000000"/> ↓ parse_char_property() CharProperty( id=0, height=1000, text_color="#000000", font_ref=CharFontRef(hangul=0, latin=0, ...), bold=True, underline=CharUnderline(type="BOTTOM", shape="SOLID", color="#000000"), ) ↓ serialize_char_property() <hh:charPr id="0" height="1000" textColor="#000000"> <hh:fontRef hangul="0" latin="0" .../> <hh:bold/> <hh:underline type="BOTTOM" shape="SOLID" color="#000000"/> </hh:charPr>

Target Pattern: ParagraphProperty

ParagraphProperty (header.py lines 359-377) has 7 dedicated sub-dataclasses and 7 parse_*() functions.

CharProperty will follow the same pattern with 9 sub-dataclasses and 9 parse_*() + 10 serialize_*() functions.
