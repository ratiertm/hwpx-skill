## 디자인 원칙

### 핵심

1. **주제에 맞는 색상을 선택한다** — 파란색 디폴트 금지. "이 색상을 완전히 다른 문서에 넣어도 어울린다면, 충분히 구체적인 선택이 아닌 것"
2. **매 섹션마다 시각 요소가 있어야 한다** — 텍스트만 있는 섹션은 잊혀진다
3. **같은 레이아웃을 반복하지 않는다** — 표→목록→박스→인용문 순환
4. **60-30-10 법칙** — 주색 60%, 보조색 30%, 악센트 10%

### Before Starting — 문서 생성 전 결정

1. **주제에 맞는 팔레트 선택** (아래 10종 중 선택 또는 커스텀)
2. **문서 성격 파악** — 공문서(절제)/보고서(데이터 중심)/제안서(설득)/안내문(친근)
3. **시각 모티프 결정** — 하이라이트 박스? 아이콘 표? 컬러 구분선? 하나를 골라 반복
4. **표지 필요 여부** — 3페이지 이하면 표지 불필요

## 주제별 색상 팔레트

문서 주제에 맞는 팔레트를 선택한다. Administrative Slate가 기본이지만, 주제가 명확하면 다른 팔레트를 사용.

| **Forest & Growth** | `#2C5F2D` | `#97BC62` | `#F5F5F5` | `#F9FBF7` | 환경, ESG, 성장 전략 |
| **Warm Executive** | `#B85042` | `#E7E8D1` | `#A7BEAE` | `#FBF9F7` | 제안서, 브랜딩 |

| **Charcoal Minimal** | `#36454F` | `#F2F2F2` | `#212121` | `#FFFFFF` | 미니멀, 기술 문서 |

### 팔레트 적용 방법

```python
from pyhwpxlib import HwpxBuilder

# 팔레트 정의 (문서 시작 전 결정)
PALETTE = {
    'primary': '#2C5F2D',       # Forest — 헤더, 섹션 제목
    'on_primary': '#FFFFFF',    # primary 위 텍스트
    'secondary': '#97BC62',     # 보조 — 하이라이트 박스
    'accent': '#F5F5F5',        # 악센트 — 콜아웃
    'surface': '#F9FBF7',       # 배경
    'on_surface': '#2b3437',    # 본문 텍스트
    'on_surface_var': '#586064',# 메타데이터
}

doc = HwpxBuilder(table_preset='corporate')

# 섹션 제목 — primary 색상
doc.add_paragraph('1. 서론', bold=True, font_size=14, text_color=PALETTE['primary'])

# 하이라이트 박스 — secondary 배경
doc.add_table([['핵심 요약']],
    cell_colors={(0,0): PALETTE['secondary']}, header_bg='',
    cell_styles={(0,0): {'text_color': PALETTE['primary']}},
    row_heights=[2400], cell_margin=(400,400,300,300), use_preset=False)

# 표 헤더 — primary 배경
doc.add_table([['항목', '값'], ['A', '100']],
    header_bg=PALETTE['primary'])
```

## 레이아웃 패턴

### 섹션별 시각 요소 매핑

| 내용 유형 | 레이아웃 패턴 | HwpxBuilder 구현 |
|----------|-------------|-----------------|
| **핵심 수치** | 큰 숫자(24pt) + 작은 설명(10pt) | `add_paragraph(수치, font_size=24, bold=True)` + `add_paragraph(설명, font_size=10)` |
| **요약/결론** | 하이라이트 박스 (배경색) | `add_table([[텍스트]], cell_colors=...)` |
| **비교 데이터** | 2열 표 + 색상 대비 | `add_table(..., col_widths=[21260, 21260])` |
| **순서/프로세스** | 번호 목록 + 들여쓰기 | `add_numbered_list(items)` |
| **인용/참고** | 콜아웃 박스 (보라/파스텔) | `add_table([[텍스트]], cell_colors={(0,0): tertiary})` |
| **주의/경고** | accent 색상 텍스트 | `add_paragraph(텍스트, text_color=accent, bold=True)` |
| **타임라인** | 번호 + 날짜 강조 | `add_numbered_list` + 날짜는 bold |
| **인물/조직** | 키-값 표 (라벨 볼드) | `add_table(..., header_bg='', cell_styles={...})` |
| **장문 본문** | 2칸 들여쓰기 + 소제목 | `add_paragraph('  텍스트...', font_size=11)` |

### 섹션 구성 순환 패턴

같은 패턴이 연속하지 않도록 순환:

```
섹션1: 제목 + 하이라이트 박스 + 본문
섹션2: 제목 + 표 (데이터)
섹션3: 제목 + 본문 + 글머리표 목록
섹션4: 제목 + 콜아웃 박스 + 본문
섹션5: 제목 + 큰 수치 + 설명
```

### 대비 구조 (Sandwich)

긴 문서에서 시각적 리듬:
```
표지 — 어두운 배경 (primary 색상)
본문 — 밝은 배경 (surface)
결론 — 어두운 배경 또는 하이라이트 박스
```

## 타이포그래피

### 폰트 체계

| 요소 | 크기 | 스타일 | 색상 |
|------|------|--------|------|
| 문서 타이틀 | 18pt | bold, CENTER | primary |
| 섹션 제목 | 14pt | bold | primary |
| 본문 | 11pt | normal, 2칸 들여쓰기 | on_surface |
| 부제/캡션 | 10pt | normal | on_surface_var |
| 참고/미주 | 8pt | normal | outline_var |
| 큰 수치 | 24pt | bold | primary 또는 accent |
| 표 헤더 | 11pt | bold | on_primary (흰색) |

### 한글 폰트 선택

| 문서 유형 | 본문 | 제목 | 분위기 |
|----------|------|------|--------|
| 공문서/정부 | 바탕 | 바탕 Bold | 공식적, 전통적 |
| 기업 보고서 | 함초롬돋움 | 함초롬돋움 Bold | 현대적, 깔끔 |
| 학술 논문 | 바탕 | 바탕 Bold | 학술적, 진지 |
| 마케팅/제안 | 맑은 고딕 | 맑은 고딕 Bold | 세련, 친근 |

## 시각 요소 규칙

### 매 섹션 최소 1개

텍스트만 있는 섹션은 금지. 다음 중 하나를 반드시 포함:
- 표 (데이터/비교/키-값)
- 하이라이트/콜아웃 박스
- 글머리표/번호 목록
- 큰 수치 + 설명
- 구분선 (텍스트 방식)
- 이미지 (있는 경우)

### 색상 사용 비율

```
Primary:   60% — 헤더, 제목, 표 헤더 배경
Secondary: 30% — 하이라이트 박스, 정보 박스
Accent:    10% — 경고, 강조, 콜아웃
```

### 간격

| 요소 | 간격 |
|------|------|
| 제목 → 본문 | 빈 줄 없음 (바로 아래) |
| 섹션 → 섹션 | 빈 줄 1개 |
| 표 전후 | 빈 줄 1개씩 |
| 박스 전후 | 빈 줄 1개씩 |
| 문서 끝 주석 | 빈 줄 2개 |

## 피해야 할 실수

1. **같은 레이아웃 반복** — 표→표→표 금지. 표→목록→박스→인용문 순환
2. **파란색 일색** — 주제와 무관한 기본 파란색 사용 금지. 주제에 맞는 팔레트 선택
3. **텍스트만 있는 섹션** — 최소 1개 시각 요소 (표/박스/목록)
4. **모든 문서에 같은 표지** — 표지가 필요한지 먼저 판단. 3페이지 이하면 불필요
5. **순검정(#000000) 사용** — 반드시 on_surface(#2b3437 등) 사용
6. **과도한 page_break** — 자연스러운 흐름 우선. 주제 대전환에서만 사용
7. **원문 요약/재작성** — 원문 텍스트 그대로 사용. LLM은 배치와 스타일만 담당
8. **폰트 크기 점프** — 18→14→11→10→8 점진적 축소 유지

## QA 프로세스

### 생성 후 필수 검증

```
1. pyhwpxlib validate output.hwpx  ← 구조 검증
2. Whale/한컴오피스에서 열기        ← 시각 검증
3. 체크리스트:
   □ 텍스트가 잘리거나 넘치지 않는가?
   □ 표 열 너비가 내용에 맞는가?
   □ 색상 대비가 충분한가? (밝은 배경에 밝은 텍스트 없는지)
   □ 같은 레이아웃이 연속 반복되지 않는가?
   □ 모든 섹션에 시각 요소가 있는가?
   □ page_break가 적절한가?
4. 수정 → 재검증 반복
```

### 시각 검증 요청 (사용자에게)

문서 생성 후 반드시 물어본다:
```
"Whale에서 열어 확인해주세요. 수정할 부분이 있으면 알려주세요."
```
